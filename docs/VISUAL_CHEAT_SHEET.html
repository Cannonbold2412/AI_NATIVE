🚀 CONXA VISUAL CHEAT SHEET

Complete System Reference — Print & Keep Handy

📊 THE 7-STAGE PIPELINE
📹 Record → ⚙️ Pipeline → 🧠 LLM → ✏️ Edit → 🔧 Compile → 📦 Package → ⚡ Execute
📹 Recording
What Happens:
  • Playwright opens real Chrome
  • JavaScript bridge captures all interactions
  • Screenshots taken for each action
  • Events stored in events.jsonl
Output Size: 50-100 KB per event
⚙️ Pipeline
5 Processing Stages:
  • Validate - Schema check
  • Clean - Normalize text
  • Dedupe - Merge scrolls
  • Enrich - Add metadata
  • Annotate - Scroll delta
Time: ~200ms per event
🧠 LLM Enrichment
Semantic Extraction:
  • Extract intent from element text
  • Normalize text to lowercase
  • Compute confidence score (0.0-1.0)
  • Detect input type (email, password)
Model: Claude Opus 4.7 Timeout: 2 seconds
✏️ Human Editing
What Humans Refine:
  • Verify LLM intent (submit_form?)
  • Pick best selector (ID > class > text)
  • Adjust timing (wait_after, timeout)
  • Add validation rules (success criteria)
Why: Improve reliability & recovery
🔧 Compilation
Conversion Process:
  • Event → Action object
  • Score selectors (0.0-1.0)
  • Rank: Best + 3 fallbacks
  • Validate intent consistency
  • Resolve dependencies
Time: ~500ms
📦 Package
Distribution Files:
  • README.md - Skill descriptions
  • skills.json - All skills
  • execute.py - Executor
  • recovery.py - LLM recovery
  • vision_recovery.py - Vision layer
Publishable to: GitHub

📊 Selector Scoring

Type Score Stability Example
ID-based CSS 0.95 ⭐⭐⭐⭐⭐ Excellent button#submit
Class-based CSS 0.75 ⭐⭐⭐ Good button.btn-primary
Attribute CSS 0.70 ⭐⭐⭐ Good button[type='submit']
XPath 0.65 ⭐⭐ Fragile //button[@id='submit']
Text-based 0.45 ⭐ Very Fragile //button[text()='Submit']

🛡️ 4-Layer Self-Healing Recovery

L1: Selector Context
Method: Try alternatives
primary → alternatives

⏱️ Timeout: Instant
✅ Success: 60-70%
L2: Text Anchors
Method: Search with priority
"Submit" (1.0) → "Send" (0.8) →...

⏱️ Timeout: Instant
✅ Success: 50-60%
L3: Text Variants
Method: Normalized search
lowercase, plurals, role-based

⏱️ Timeout: Instant
✅ Success: 40-50%
L4: Vision Recovery
Method: Vision LLM analysis
Detect + intent-match candidates

⏱️ Timeout: 120 seconds
✅ Success: 75-85%
📊 Combined Strategy: Layers 1-3 (instant) handle 80% of failures. Layer 4 (vision) catches remaining 20%. Total recovery success: ~95% of all errors.

Retry Policy: Element actions (click, fill) = 2 attempts | Navigate/Check = 3 attempts

📋 Key Data Structures

Raw Event (Post-Recording)
{ "action": { "action": "click", "button": "left" }, "target": { "tag": "button", "inner_text": "Submit", "classes": ["btn", "primary"] }, "selectors": { "css": "button.btn-primary", "xpath": "//button" }, "semantic": { "normalized_text": "submit", "llm_intent": null }, "visual": { "full_screenshot": "path.jpg" } }
Compiled Action (Post-Compilation)
{ "id": "step_1", "action": "click", "selector": "button#submit", "selector_confidence": 0.95, "selectors_fallback": [ "button.btn-primary", "//button[@type='submit']" ], "validation": { "success_criteria": [ {"type": "url_contains", "value": "/dashboard"} ] }, "wait_after_ms": 2000 }

✅ Best Practices

Selectors

Validation

Timing

❌ DON'T
Error Handling
Don't record error paths
Credentials
Don't hardcode passwords
Text Content
Don't rely solely on text
Complex Logic
Don't create intricate flows

🐛 Common Errors & Solutions

Error Cause Solution
Element not found Selector mismatch Check screenshot, use fallback, trigger recovery
Ambiguous selector Multiple matches Add specificity (ID, data-test)
Validation failed Expected state missing Increase wait time, verify expectation
LLM timeout Vision model slow Use structured selector, skip enrichment
Circular dependency Actions depend on each other Reorder actions, split into skills

🎯 Key Takeaways
  • Event: Raw user interaction (click, type, navigate)
  • Action: Compiled executable step with fallback selectors
  • Skill: Sequence of related actions that can be executed
  • Intent: Semantic meaning (submit_form, fill_field) extracted by LLM
  • Recovery: 4-layer tiered system (L1-L3 instant selectors, L4 vision 120s) achieves ~95% success
  • Confidence: Every action has selector confidence + validation rules
  • Resilience: Automatic recovery handles UI changes without re-recording
